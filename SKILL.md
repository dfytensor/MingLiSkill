## 一、用户问答模式（日常使用）

当用户问命理问题时，agent 的工作流程：

### 步骤 1：从用户消息中提取信息

从用户的自然语言中提取：
- **出生信息**：年、月、日、时、性别
- **问题类别**：事业/婚姻/财运/健康/子女/学业/家庭/性格/官非/灾劫/外貌/运势
- **具体问题**：用户问了什么
- **选项**（如有）：A/B/C/D 各选项文本

如果用户没有给出选项，agent 可以直接根据排盘数据回答。

### 步骤 2：调用 `analyze_question` 获取排盘数据

```python

    year=1974, month=4, day=28, hour=16, gender='男',

        {"letter": "C", "text": "技术工程师"},

data = json.loads(result)
```

## 二、推理指南

### 2.0 系统性推理框架（★ 必须遵循，在分类推理之前先过这6条）

以下6条规则是对所有类别通用的推理纪律。**任何推理步骤都必须受这些规则约束。**

---

#### 规则1：日主先行锚定原则

任何判断的第一锚点是日主天干五行本质，后续所有十神/紫微分析都是在这个底色上叠加。

| 日主 | 本质 | 职业底色 | 性格底色 | 身体底色 |
|------|------|----------|----------|----------|
| 甲 | 栋梁之木，正直向上 | 管理、教育、公职 | 正直、仁德 | 高大 |
| 乙 | 藤萝之木，柔韧攀附 | 艺术、协调、辅助 | 温柔、灵活 | 瘦秀 |
| 丙 | 太阳之火，光明热烈 | 政治、演艺、能源 | 热情、开朗 | 红润、健壮 |
| 丁 | 灯烛之火，文明细腻 | 文化、技术、精密 | 细腻、敏感 | 中等偏瘦 |
| 戊 | 城墙之土，厚重稳健 | 地产、金融、稳定岗 | 稳重、守信 | 敦厚结实 |
| 己 | 田园之土，滋养包容 | 服务、教育、农业 | 温和、包容 | 圆润 |
| 庚 | 斧钺之金，刚毅决断 | 军警、运动、机械、金融 | 刚毅、果断 | 筋骨强健 |
| 辛 | 珠玉之金，精致贵气 | 珠宝、设计、金融、法律 | 精致、好面子 | 白皙纤细 |
| 壬 | 江河之水，奔流不息 | 贸易、物流、旅游 | 豪爽、自由 | 圆润 |
| 癸 | 雨露之水，渗透智慧 | 研究、策划、咨询 | 聪明、细腻 | 柔韧 |

**使用方式**：无论判什么类别，先读出日主天干，以此确定人物的"底层操作系统"，再往上叠加十神和紫微。

★ **日主体型 = 五行底色 × 旺衰系数（v4修正）**：
上表中的"身体底色"是日主五行**中等旺衰**下的基准体型。实际体型需根据旺衰程度调整：

**★ v4关键纪律：体型判断必须以 huoyuan_analysis 为准，禁用 bazi 的日主强弱标签**。原因：bazi 的"身强/身弱"是粗粒度标签，huoyuan_analysis.旺衰 更精细（中和偏旺/中和偏弱/身弱/身强），且包含得令信息。体型需要精确的旺衰系数。

- **得令=true + 旺衰为身强/中和偏旺** → 体型向"大一号"方向调整。如己土得令偏旺 → 可高大敦厚；甲木得令偏旺 → 更加高大魁梧
- **得令=false + 旺衰为身弱/中和偏弱** → 体型向"小一号"方向调整。如庚金失令偏弱 → 偏瘦削；**甲木失令偏弱 → 不再高大，偏向中等偏矮**（★ 这是关键：甲木虽底色高大，但失令+偏弱足以拉低到中等偏矮）
- 得令=false + 旺衰为中和偏旺 → 体型不变（本色）
- 日主五行占比>35% → 体型强化一个等级
- 日主五行占比<15% → 体型弱化一个等级

★ **财星与体型的关系（v4新增 — 判胖瘦的第二维度）**：
财星五行会影响体型胖瘦，需与日主体型叠加判断：
| 财星五行 | 对体型的影响 | 原因 |
|---------|------------|------|
| 土财旺(>2.0) | **倾向丰满/肥胖** | 土主肉，财旺肉多 |
| 金财旺(>2.5) | 倾向结实/骨感 | 金主骨，财旺骨骼突出 |
| 水财旺(>2.5) | 倾向圆润/水肿型 | 水主润下 |
| 木财旺(>2.5) | 倾向瘦长 | 木主筋，拉伸体型 |
| 火财旺(>2.0) | 倾向红润/中等 | 火主血脉 |

★ 财星旺 + 日主得令偏旺 → 可同时高大+丰满。财星旺 + 日主失令偏弱 → 矮小+丰满/肥胖。

#### 规则2：地支 > 天干（内外分层原则）

- **天干** = 外在表现、别人看到的、表面性格
- **地支（含藏干）** = 内在真实、潜意识、实际处境
- ★ 当天干和地支信号冲突时，**地支为真，天干为表**。一个人天干正印（表面知书识礼）+ 地支双七杀（内在恐惧压抑）→ 内在恐惧是核心性格，正印只是外包装。
- ★ 克身之星（七杀/正官）出现在**月支或日支**（贴身位置），对性格的影响权重大于天干所有十神。计算克身之星的地支数量：≥2 = 压抑/恐惧/压力主导。

---

#### 规则3：制化改写原始十神（化学方程式原则）

十神不是孤立存在的，它们之间会发生化学反应。**有制化的十神，性格表现被改写**：

| 原始十神 | 制化关系 | 改写后的性格 |
|----------|----------|--------------|
| 七杀 | 被食神所制 | 不暴戾，反而是"聪明有才、能化压力为动力" |
| 七杀 | 被正印所化 | 不攻击性，而是"稳重威严、以德服人" |
| 七杀 | 无制无化 | 原始暴力/压迫/恐惧，不可控攻击性 |
| 伤官 | 被正印所制 | 创造力被引导，不叛逆反而"有创见的学者" |
| 伤官 | 无制 | 叛逆、不守规矩、攻击权威 |
| 劫财 | 见正官 | 冲动被约束，反而"有纪律的竞争者" |
| 偏印 | 见食神 | 多疑被化解，变为"独特的创造性思维" |

★ 判断制化时，只看天干对天干、地支对地支（干支之间不跨层次制化，除非天干通根）。
★ 贴身的制化（月干制年干=远，月干制时干=远；食神在月支制年支七杀=远不如制月支七杀贴身有效）。

---

#### 规则4：谱系思维（禁止二分法）

所有命理判断不存在"有/无"或"高/低"的二分，必须是连续谱。

- **印星数量判学历时**：0个印星 → "学历基础弱"（不是"没学历"），需留至少一级缓冲。如印星=0 → 中学打底，再看格局是否抬升。
- **比劫数量判层次时**：比劫多(≥4个) ≠ "低层次"，它是一把双刃剑——可以是体力劳动者，也可以是精力充沛的领导者，取决于是不是有强格局星配合。
- **财星判贫富时**：财星力量只是"可能性"，不是"必然性"。身弱财旺 = "见财难得"而非"穷"。
- **所有判断句式**：必须以"倾向于...""在...基础上偏..."而非"就是..."。

---

#### 规则5：多信号交叉验证（最少2源原则）

任何选项判断必须至少有**2个独立信号源**支撑。单一信号作出的判断，置信度低，应避免。

信号源分类：
1. 八字节（日主天干五行本质、十神组合、格局类型、五行力量分布）
2. 紫微斗数（命宫主星、相关宫位主星）
3. 流年数据（liunian 标签、冲合关系、大运背景）
4. 分类分析字段（marriage_analysis、education_analysis、career_analysis 等）

**交叉验证方法**：对每个选项列出支撑它的信号和反对它的信号，取净正向最多的选项。

---

#### 规则6：主星优先于辅星（正>偏原则）

当同类十神同时出现时，正神优先级高于偏神：

| 领域 | 正神 | 偏神 | 含义 |
|------|------|------|------|
| 婚姻（女） | 正官（丈夫） | 七杀（情夫/次选） | 婚期多候选时，正官出现 > 七杀出现 |
| 婚姻（男） | 正财（妻子） | 偏财（情人/次选） | 婚期多候选时，正财出现 > 偏财出现 |
| 学业 | 正印（正统学历） | 偏印（偏门技术） | 正印对学历权重 > 偏印 |
| 事业 | 正官（体制内正职） | 七杀（创业/副职） | 正官对稳定职业权重 > 七杀 |

---

#### 规则7：十神力量权重原则（禁止按个数判断）

★★ **所有涉及数量的判断不得仅数"出现次数"，必须同时乘以对应五行力量。**

| 判断场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| 子女数 | 数官杀+食伤出现次数(如:申藏壬+丑藏癸=2子) | 查对应十神五行力量(如水1.0/9.6%=子女缘薄) |
| 配偶数 | 数财星/官星出现次数 | 综合财/官五行力量+配偶宫藏干+夫妻宫主星 |
| 学历层 | 只数印星数量 | 印星力量×格局乘数(规则见2.2学业) |
| 财帛层 | 只数财星出现次数 | 财星五行力量×占比×喜忌 |
| 灾祸轻重 | 只查有无冲刑 | 冲刑+克身星力量+是否有印制化 |

★ **力量值<1.0 的十神即使出现多次，实际影响力极弱，不可作为主信号。**
★ **力量值>4.0 的十神即使只出现1次，影响力压倒一切其他十神。**

★ **流年事件打折规则（v3新增 — 防止原局弱星遇流年旺星误判强事件）**：

当判断流年具体事件（结婚/生育/升职/发财/灾祸等）时，流年十神信号的强度必须乘以原局对应十神五行力量的打折系数：

| 原局对应十神五行力量 | 打折系数 | 事件强度 | 典型示例 |
|---------------------|---------|---------|---------|
| <0.5 | ×0.2 | 极弱/不发生 | 食伤0.3→流年伤官旺=难孕 |
| 0.5 ~ 1.0 | ×0.5 | 弱化事件 | 食伤0.8→流年伤官旺=怀孕但流产 |
| 1.0 ~ 2.0 | ×1.0 | 正常事件 | |
| 2.0 ~ 4.0 | ×1.5 | 强化事件 | |
| >4.0 | ×2.0 | 极强事件 | |

★ **操作步骤**：
1. 识别流年透干的十神类型（如2019戊戌=伤官→子女/生育事件）
2. 查该十神在原局的五行力量（如原局食伤=土=0.8）
3. 按上表打折（0.8 → ×0.5 → 弱化事件）
4. 再判断最终事件强度：弱化 → 倾向"怀孕但流产/考学失利"而非"成功生子/考取文凭"

★ **此规则覆盖所有事件的流年判断，不限于子女/生育**。判财运时：原局财星0.5 → 流年财旺 = 有小财非大财。判官非时：原局官杀2.5 → 流年官杀旺 = 实职晋升而非虚名。

---

#### 规则8：十神喜忌偏置规则（★ 所有信号必经此关）

**任何十神信号在作为判断依据前，必须检查其对应五行是否为喜用神。**

十神的作用方向取决于其五行与日主喜忌的关系：

| 十神 | 五行=喜用时的作用 | 五行=忌神时的作用 |
|------|-----------------|-----------------|
| 印星 | 帮身、助学、贵人 | **印多压身**，反而不利学业/事业 |
| 官杀 | 权力、贵气、自律 | 官非、压力、灾祸（非权力） |
| 财星 | 财富、收入、机会 | 见财有灾、破财、身弱不担财 |
| 食伤 | 才华、技艺、创造力 | 口舌是非、叛逆冲动 |
| 比劫 | 帮手、合作、竞争有利 | 争财、破财、朋友拖累 |

★ **关键示例**：
- 印星4个但印的五行(金)为忌神 → **印多不是好事**，反而压制日主，学历/事业不佳
- 财星3.3(旺)但日主身弱+财为忌 → **见财难得**，穷命
- 七杀透干但七杀五行(火)为喜用 → 可以化压力为权力（仅在身强时）

#### 规则9：印星透干加权规则（★ 禁止直接用 education_analysis 印星总数）

**印星的有效力量 ≠ 印星出现次数。必须按透干程度加权后重新计算。**

| 印星出现形式 | 正印权重 | 偏印权重 |
|-------------|---------|---------|
| 天干透出 | ×1.0 | ×0.7 |
| 地支本气 | ×0.7 | ×0.5 |
| 地支中气/余气 | ×0.4 | ×0.3 |

**有效印星力量 = Σ(各印星五行力量 × 权重)**

将加权后有效力量用于学历基础表：
- 有效力量≥3.0 → 按印星3+级处理
- 有效力量 1.5-3.0 → 按印星2级处理
- 有效力量 0.5-1.5 → 按印星1级处理
- 有效力量 <0.5 → 按印星0处理（谱系底线≠0，仍然至少中学）

### 2.1 八字推理核心

#### 判断日主强弱

- 田宅宫有天府/太阴/紫微 → +1分（房产/祖业）

**综合定等**：
- 总分≥4 → **富裕**（大富贵）
- 总分 1-3 → **小康**（正常偏上）
- 总分 -1 到 0 → **贫困偏下**
- 总分 ≤-2 或 父星缺失(孤儿信号) → **赤贫/孤儿**

★ 注意：天府在命宫不等于出身好——天府代表的是"后天积累和管理能力"，与先天家境无直接关系。判断家境只看年柱+父母宫+父母星。
★ **父母宫主星优先级纪律（v2修正）**：判断父母关系时，优先级为：**年柱喜忌 > 父母星存在性 > 父母宫主星**。年柱干支皆为喜用 → 即使父母宫主星为七杀/破军，也只是"父亲管教严格"，不代表父母关系差/吵闹/离异。父母宫主星单独不能作为判断父母关系的主要信号——必须先看年柱喜忌。

#### 子女判断（★ 力量优先，禁止数数）

- ★ 男命子女看官杀(水)五行力量和时柱食伤力量，**不按出现次数计算子女数**
- 子女相关十神力量<1.5 → 子女缘薄，倾向于少子或无子
- 紫微子女宫空宫 OR 子女宫主星弱(天机/巨门带煞) → 子女缘薄
- 子女宫主星强(紫微/天府/武曲) + 十神力量>2.0 → 子女多
- ★ 食伤被合(如巳申合=时支伤官被合)会削弱子女数量，实际可减半

#### 生育事件判断（★ v3新增 — 专用于"何时生子/流产/开刀生子"等问题）

当问题涉及具体年份的生育事件（而非"有几个子女"）时：

1. **先查原局子女星底子**：根据规则7流年事件打折表，原局食伤/官杀五行力量<1.0 → 流年生育事件打折
2. **男命生育看官杀(水)力量 + 时柱食伤**；女命生育看**食伤(子女星)力量 + 时柱食伤**
3. **流年子女星透干 + 合动时柱(子女宫)** → 该年有生育事件
4. **★ 力量打折判断**：
   - 原局食伤<0.5 → 流年食伤旺也极难成功生育（×0.2=几乎不发生）
   - 原局食伤 0.5-1.0 → 流年食伤旺但打折（×0.5）→ 有孕但倾向**流产**而非成功分娩
   - 原局食伤>1.0 → 流年食伤旺=正常生育事件
   - 若选项中同时有"开刀生子"和"流产"，原局食伤<1.0 → 流产优先
5. **金旺+生育结合**：若原局金>3.5且流年金再旺，同时流年有子女星信号 → 可判"剖腹产/开刀生子"（金=刀+子女星=生）。但前提是原局食伤>1.0

★ **关键纪律**：不要只看流年子女星旺就判"成功生子"——必须先查原局食伤五行力量。原局底子薄(<1.0)则流年信号打折。

#### 健康/抑郁判断（★ 七杀克身程度排序框架）

七杀导致抑郁/精神问题的严重程度排序(从最重到最轻):

| 优先级 | 条件 | 抑郁程度 | 说明 |
|--------|------|---------|------|
| 1 | 七杀透干 + 财星生杀(无印制) | ★★★★★ 极重 | 财生杀旺，直接克身，无化解 |
| 2 | 七杀透干 + 无制化 | ★★★★ 重 | 七杀直接克身 |
| 3 | 七杀在支贴身(月支/日支) + 财生杀 | ★★★ 中重 | 内在持续压力 |
| 4 | 七杀在支贴身 + 无制化 | ★★ 中 | 内在慢性压力 |
| 5 | 七杀透干 + 有印制化 | ★ 轻 | 压力被转化 |
| 6 | 七杀在支 + 有食神制化 | ☆ 轻微 | 压力被释放 |

★ **流年判断时**：对比每年"七杀克身净程度" = 七杀出现方式得分 - 制化得分。取最高分年份。
- 流年天干为七杀 = +3, 七杀在地支藏干 = +1
- 流年天干为财星(生杀) = +1, 流年天干为正印(化杀) = -1
- 大运天干为财(生杀) = +1, 大运天干为印(化杀) = -1
- 大运+流年皆为忌神 = +1 (叠加负面)

★ **急伤/慢病区分框架（v2新增 — 防止七杀出现一律判手术）**：

流年七杀透干 ≠ 必然手术/外伤。必须区分急性伤害和慢性失调：

| 判断维度 | 倾向急性伤害(手术/意外) | 倾向慢性失调(肥胖/肠胃/肝火) |
|---------|----------------------|---------------------------|
| 原局克身星情况 | 原局少或无克身星，突然七杀来克(冲击性) | 原局某种五行已偏旺/偏弱(长期失衡) |
| 五行偏枯程度 | 某五行=0突然出现(如金=0+流年金克木) | 某五行持续偏旺(>40%)或偏弱(<10%)数年 |
| 流年冲合位置 | 七杀冲日柱/时柱(自身/行动) | 流年合旺/合动偏旺五行 |
| 紫微疾厄宫 | 廉贞/破军/七杀(创伤/手术型) | 天梁(慢性病)、天同(慢性虚弱) |
| 健康分析信号 | 最弱五行恰好是克身星所在五行 | 最旺五行对应脏腑持续偏旺 |

**判断流程**（★ 严格按顺序执行，上一条触发后不再执行下一条）：
1. 先读 `health_analysis`，确认各五行状态
2. ★ **慢性无条件优先规则（v3新增 — 急/慢信号冲突裁决）**：
   - 若原局**任意**五行偏旺(>40%) → **无论是否同时存在"某五行=0+流年克身"急性信号，慢性无条件胜出**
   - 不需要"流年继续生旺"条件——偏旺>40%本身就代表持续体质失衡倾向
   - ★ 即使流年七杀透干+原局该五行=0，只要>40%偏旺存在，仍判慢性而非手术
   - 慢性问题的选项特征：多脏腑受累、持续数年、"反复""困扰""肥胖""变胖"等描述
   - 偏旺五行对应的脏腑即为问题核心（火>40%→心/代谢/肥胖；木>40%→肝/胆；土>40%→脾胃/肥胖；金>40%→肺/皮肤；水>40%→肾/水肿）
3. 若原局**无**>40%偏旺五行 + 某五行=0且流年该五行透干克身 + 原局无制化 → 可判急性伤害/大型手术
4. 若原局无>40%偏旺 + 某五行=0但流年有制化（食神/正印所在天干可制克身星）→ 急性伤害可缓解，非大型手术

★ **关键纪律**：只要原局存在>40%偏旺五行，停止急性伤害推理，直接进入慢性体质判断。七杀透干只决定慢性问题的**严重程度**（加重体质问题），不改变慢性本质。

### 2.4 常见推理陷阱

以下是从实际测试中总结的推理易错点，agent 在推理时务必注意：

| # | 陷阱 | 正确做法 |
|---|------|---------|
| 1 | **只看印星数量判学历** | 必须综合格局（官印相生、杀印相生即使印少也可高学历）、官禄宫主星、大运走势。比劫多≠低学历。 |
| 2 | **被"忌神年"标签误导婚期** | 流年标签只是参考。流年地支藏干出现配偶星=婚期第一信号，即使该年被标"忌神"，以十神为准。 |
| 3 | **忽略配偶星直接出现** | 判断婚期时优先级：流年配偶星出现 > 日支被合 > 流年天干透星。不要只追"冲合"而忽略更直接的十神信号。 |
| 4 | **家境只盯年柱财星** | 综合看：年柱喜忌 + 财/印是否有根得令 + 月柱父母宫状态。财星坐忌地被泄 = 偏贫。 |
| 5 | **忽略格局层次** | "官印相生""杀印相生"等高层次格局本身就暗示较强的成就潜能，不能仅看单一十神数量。 |
| 6 | **忽略大运背景** | 流年判断必须结合当时所在大运。大运为喜用+流年为喜用=大吉；大运为忌+流年为忌=大凶。工具返回的 `liunian` 数据已包含大运信息。 |
| 7 | **只看天干忽略地支（外表误判内心）** | ★ 天干=外表、地支=内在真实。性格判断中，地支克身之星（七杀/正官）数量≥2时，压抑/恐惧为主导向，天干正印等光鲜十神只是外包装。身强标签不代表不恐惧。 |
| 8 | **正神/偏神等权重混用** | 男命婚期：正财(正妻) > 偏财(偏妻)。女命：正官 > 七杀。学业：正印 > 偏印。不可把正偏等权重处理。 |
| 9 | **忽略十神制化关系** | ★ 七杀被食神所制 = 不暴戾反而聪明有才。七杀被印制 = 稳重威严。无制无化 = 原始暴戾。判断配偶/性格时必须检查克身之星是否有制化。 |
| 10 | **比劫重直接判低层次** | 比劫极旺(≥4.0) + 紫微/武曲/天府在命宫或官禄 = 格局层次跳级。精力充沛+强执行力 ≠ 体力劳动者。 |
| 11 | **只看单一十神判职业（刻板联想）** | 事业判断必须：日主五行底色 → 格局方向 → 官禄宫主星。只靠偏印多就判玄学、只靠食伤就判艺术 = 不可靠。 |
| 12 | **二分法思维（0=无/多=极高）** | 印星0 ≠ 文盲（至少中学）。比劫多 ≠ 一定低层次。所有判断是连续谱，句式用"倾向于...程度"而非"就是..."。 |
| 13 | **单一信号做判断** | 每个答案必须至少有2个独立信号源支撑。只有一个信号(如只看七杀就判家暴) → 置信度低，应降低权重或放弃。 |
| 14 | **错把后天宫星当先天判断** | ★ 天府在命宫≠出身好。天府代表"后天积累/管理能力"，判断家境应看年柱+父母宫+父母星。同理，紫微在命≠出身高贵。 |
| 15 | **流年干支张冠李戴** | ★ 必须从 `liunian.年份流年对比` 逐字读取每年干支。禁止从其他年份的数据中推测/拼凑/记忆错误。错读干支=整题必错。 |
| 16 | **子女数靠数个数** | 子女数不能按"官杀出现次数"统计。必须看: 官杀五行力量(水>2才算有力) + 紫微子女宫 + 食伤是否被合/被冲。水<1.0 → 子缘极薄。 |
| 17 | **忽视格局乘数效应** | 比劫极旺(>4.0) + 紫微/武曲在命宫或官禄 = 格局翻倍，学历跳2级。这是硬规则，印星少但满足此条件也可到博士。 |
| 18 | **七杀克身不比较流年** | 判断"最严重年份"必须逐年对比七杀出现方式(透干 vs 在支)、是否财生杀(恶化)、是否有印制化(缓解)、大运背景(叠加)。选净克力最高年。 |
| 19 | **年柱喜忌不区分就判家境** | 年柱天干地支的十神必须确认是否为喜用神。年柱全是忌神→贫；一喜一忌→中等偏下。不能因年柱有"印星"就判有家底——忌神印不帮助出身。 |
| 20 | **偏印≥3无条件覆盖日主底色** | ★ 偏印为忌神时才覆盖日主；偏印为喜用帮身时强化日主本身特质（庚+偏印喜用→刚毅运动型而非艺术型）。先看huoyuan_analysis中偏印五行是否为喜用。 |
| 21 | **日支藏印但忽略冲刑** | 日支藏印实际有效的条件是日支未被冲/刑/穿。如亥巳冲 → 巳中丙火正印被冲掉，不能算作1个印星。 |
| 22 | **婚期评分忽略大运冲命** | ★ 大运冲命（如大运酉冲年支卯）= 婚期稳定性大扣分(-3)。大运冲命+流年忌神 = -5，该年极不适合婚期。标签同时有喜用和忌神时两者抵消，按纯十神打分。 |
| 23 | **七杀流年出现一律判手术/意外** | ★ 需区分急性伤害和慢性失调。原局某五行偏旺(>40%)被流年继续生旺 → 优先慢性体质问题（肥胖/肝火/肠胃），非手术。必须先读health_analysis确认各五行状态。 |
| 24 | **用父母宫主星单独判父母关系** | 父母关系判断优先级：年柱喜忌 > 父母星存在性 > 父母宫主星。年柱全喜用 → 即使父母宫七杀，也只是"父亲管教严"而非父母关系差/离异。 |
| 25 | **日主体型不看旺衰** | 体型=五行底色×旺衰系数。己土身旺得令→可高大敦厚；庚金身弱→偏瘦削。五行占比>35%强化一级，<15%弱化一级。 |
| 26 | **十神概念偷换(偏印≠中式/传统)** | ★ 偏印=偏门/独特/怪异/创新；正印=传统/正统/保守。判断时不可将偏印等同于"中式""复古""传统"——那是正印的事。同样，食神≠艺术才华(伤官)、正官≠所有管理角色。详见十神概念锚定表。 |
| 27 | **印多为忌不检查，当作喜用判** | ★ 印星4个但印的五行(如金)为忌神→印多压身，反而不利学业/事业。必须查huoyuan_analysis中印星五行为喜用还是忌神。 |
| 28 | **年柱判断只看十神不看内部生克** | ★ 年柱丙戌(伤官丙生财星戌)=伤官生财=先天富基。年干生年支(+2分)，年干克年支(-1分)。必须看年柱内部生克关系，不能只对年干年支做喜忌判断。 |
| 29 | **自身强弱与家庭出身混判** | ★ 日主身弱不能担财 是 自身财运弱，不等于 家庭出身穷。判断家境只看年柱+父母星+父母宫，不看日主强弱。 |
| 30 | **身弱走七杀年判为"突破年"** | ★ 身弱者七杀克身=危机/压力/灾祸，不是突破。身弱突破年=喜用帮身年(印/比劫透干)。反之身强者七杀年可化压力为突破。 |
| 31 | **格局倾向未作为职业第一信号** | ★ 事业判断4步递进必须遵守顺序：格局倾向(career_analysis.格局倾向)锁定大类 → 官禄宫主星细化 → 日主底色验证 → 迁移宫+夫妻宫交叉检查。食伤生财格局=销售/业务/技术变现，不可用日主底色覆盖格局信号。 |
| 32 | **紫微跨宫交叉不足，只看官禄判职业** | ★ 夫妻宫廉贞七杀+迁移宫武曲破军 = 创伤/殡葬/死亡行业信号。迁移宫代表外部工作环境，夫妻宫代表合作关系，两者必须与官禄宫联合判断职业类型。 |
| 33 | **科系/学业专用规则被日主底色覆盖** | ★ 当question涉及"读什么科系"时，偏财>3.0+偏印≥3→艺术变现的科系专用规则优先级最高(无论偏印喜忌)，优于日主底色规则。因为旺财克偏印，帮身效应被抵消。 |
| 34 | **直接用education_analysis印星总数未加权** | ★ 印星必须按透干程度加权：天干透出正印×1.0，地支本气×0.7，中气余气×0.4；偏印×0.7/0.5/0.3。加权后再对照学历表。正印在年支中气(余气)≈0.4权重，不可算作1个。 |
| 35 | **急伤/慢病信号共存时选急伤** | ★ 原局任意五行>40%偏旺时，慢性信号**无条件优先**于急性信号。即使同时有"某五行=0+流年克身"，慢性仍胜出。七杀透干只加重慢性程度，不改变慢性本质。 |
| 36 | **流年事件不看原局十神力量** | ★ 判断流年事件(生育/升职/发财等)时，流年信号强度必须×原局对应十神五行力量系数。食伤=0.8→流年伤官旺=流产非成功生子。财星=0.5→流年财运=小财非大财。 |
| 37 | **天机官禄直接判玄学/偏门** | ★ 天机=技术/策划/灵活，不等于玄学。只有偏印≥3(忌神)+非正统格局时，天机才偏向玄学方向。其余情况天机=灵活技术/培训(如健身教练的训练方案、IT敏捷开发)。 |
| 38 | **盲目信任 family_analysis 父亲星缺失** | ★ family_analysis 是工具自动生成，可能遗漏偏财。不能仅凭 父母星位置_父亲=[] 直接套用 -3分孤儿规则。必须手动核查四柱+藏干中偏财是否真的不存在。只有手动确认偏财五行力量=0时才适用 -3分。 |
| 39 | **用 bazi.日主强弱 判体型** | ★ bazi 的"身强/身弱"是粗粒度标签，体型判断必须以 huoyuan_analysis.旺衰 + 得令 为准。甲木失令偏弱 → 不再高大，变为中等偏矮。不要因为日主标签写"身强"就以"高大"为基准体型。 |
| 40 | **命宫主星覆盖地支克身星判性格** | ★ 命宫武曲/七杀等"大胆型"主星 ≠ 此人性格大胆外向。性格判断中：地支克身星 > 命宫主星。日支七杀克身 → 内在孤僻/压抑，命宫武曲只是外包装。选项有"孤僻/沉默"和"大胆/竞争"对立时选前者。 |

## 三、Benchmark 测试模式

### 3.1 跑纯规则引擎（无 LLM，0.02 秒）

```bash
python predict_mingli.py --batch <data_path> --output result.json
```

### 3.2 Agent 逐题测试（干净测试）

**必须在全新会话中执行，确保无上下文污染。**

步骤：
1. 读取 benchmark 题目文件（如 `data.json`）
2. 随机抽取 N 题（**记录题目 ID**）
3. 对每题调用 `analyze_question`（**不要读取答案字段**）
4. 阅读排盘数据，推理出答案
5. 全部答完后，对照 `answer` 字段打分

示例脚本（生成无答案的测试题）：

```python
import json, random, sys, os
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))

with open(r'<benchmark_data_path>', 'r', encoding='utf-8') as f:
    data = json.load(f)

random.seed(<NEW_SEED>)
sampled = random.sample(data['questions'], 20)

for q in sampled:
    bi = q['birth_info']
    # 注意：不要 print q['answer']！
    print('ID: %s | Cat: %s' % (q['id'], q['category']))
    print('Birth: %s %d-%d-%d %dh' % (bi['gender'], bi['year'], bi['month'], bi['day'], bi.get('hour', 12)))
    print('Q: %s' % q['question'])
    for o in q['options']:
        print('  %s. %s' % (o['letter'], o['text']))
    # 调用工具获取排盘数据
    from tools import HybridMingliToolkit
    htk = HybridMingliToolkit()
    r = htk.analyze_question(
        year=bi['year'], month=bi['month'], day=bi['day'],
        hour=bi.get('hour', 12), gender=bi['gender'],
        category=q['category'], question=q['question'],
        options_json=json.dumps(q['options'], ensure_ascii=False)
    )
    data = json.loads(r)
    print('日主: %s(%s) %s | yong=%s' % (
        data['bazi']['日主'], data['bazi']['日主五行'],
        data['bazi']['日主强弱'], data['bazi']['喜用神']))
    print('十神: %s' % json.dumps(data['bazi']['十神'], ensure_ascii=False))
    print('命宫: %s | 官禄: %s | 夫妻: %s' % (
        data['ziwei']['命宫主星'], data['ziwei']['官禄宫主星'], data['ziwei']['夫妻宫主星']))
    print('Rules建议: %s (仅供参考)' % data['rules_suggestion']['suggested_answer'])
    print('---我的答案:___')
    print()
```

**答完所有题后**，再用脚本对照打分：
```python
answers = {
    'ftb_xxxx': 'A',  # 你推理的答案
    ...
}
with open(r'<benchmark_data_path>', 'r', encoding='utf-8') as f:
    data = json.load(f)
correct = sum(1 for q in data['questions'] if q['id'] in answers and answers[q['id']] == q['answer'])
print('正确: %d/%d = %.1f%%' % (correct, len(answers), correct/len(answers)*100))
```
