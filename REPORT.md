## 4. 测试方法

### 4.1 方法 A：纯 LLM 直接推理

LLM 直接读取题目和选项，进行链式推理后给出答案。

```
prompt = 题目 + 出生信息 + 选项
→ LLM 推理 → 答案提取（正则匹配 "答案：X"）
```

### 4.2 方法 B：纯规则引擎

纯 Python 规则引擎，根据出生信息排八字盘，用十神/五行/冲合等规则对选项打分。

```
出生信息 → 八字排盘 → 十神/五行分析 → 规则打分 → 选最高分选项
```

打分逻辑：
- **性格题**: 日主天干性格映射 + 五行旺衰性格 + 十神性格特质
- **事业题**: 十神职业映射 + 日主强弱职业倾向 + 格局分析
- **婚姻题**: 配偶星/配偶宫分析 + 冲合关系 + 流年配偶星触发
- **财运题**: 财星力量 + 比劫夺财 + 身强身弱财富水平
- **健康题**: 五行对应脏腑 + 最弱五行 + 七杀伤官健康事件
- **家庭题**: 印星（父母）+ 比劫（兄弟）+ 财星（家境）+ 流年冲合
- **年选题目**: 流年天干十神匹配 + 大运流年喜忌 + 六冲六合分析

### 4.3 方法 C：LLM + 命理工具注入

先用工具计算八字/紫微排盘数据，注入到 prompt 中，让 LLM 基于工具数据进行推理。

```
出生信息 → 工具排盘（八字+紫微+大运+流年+冲合+神煞）
→ 注入数据到 prompt → LLM 推理 → 答案提取
```

注入内容包括：
- 四柱八字、日主强弱、五行力量、十神分布
- 喜用神/忌神、空亡、纳音
- 大运分析（天干十神、与喜用神关系）
- 流年分析（天干十神、与日主关系）
- 地支冲合分析
- 神煞分析
- 分类专项分析（婚姻/事业/财运/健康/学业/性格等）

## 6. 关键发现

### 6.1 规则引擎的优势

1. **速度**: 0.02 秒 vs LLM 的 9+ 小时，快 170 万倍
2. **确定性**: 结果可复现，无随机性
3. **模式匹配强**: 灾劫（100%）、官非（100%）、健康（53%）等类别通过十神/五行关键词匹配表现优秀
4. **零成本**: 无需 GPU/API 调用

### 6.2 LLM + 工具注入的优势

1. **推理型问题**: 事业（44% vs 24%）、性格（57% vs 36%）大幅提升
2. **综合分析**: LLM 能整合多条命理线索进行推理
3. **理解语义**: 能理解选项的语义差异（如"打工"vs"创业"）

### 6.3 LLM + 工具注入的劣势

1. **模型能力限制**: qwen3.5-9b 是小模型，有时幻觉（编造选项 E）
2. **长 prompt 干扰**: 注入数据过多时超出上下文长度
3. **慢**: 每题 30-300 秒
4. **婚姻类灾难**: 20% 远低于规则引擎 33%，LLM 对婚姻年份推理能力弱

### 6.4 纯规则引擎的天花板

- 96/160（60%）题目始终错误，纯关键词匹配无法解决
- 需要解决的问题类型：
  - 婚姻年份推断（需要大运流年交互分析）
  - 职业精确匹配（需要理解行业描述语义）
  - 学历判断（需要综合正印力量+时代背景）
  - 家庭年份事件（需要多因素协同推理）

## 9. 结论

1. **混合路由排名全球第 2**：41.77%（66/158），仅次于 Tianfu Agent（45.0%），超越 Claude Opus 4.6、GPT-5.4 等所有主流大模型
